# 📊 Канбан для управления обращениями

## Что такое Канбан?

**Канбан** — это метод визуального управления работой, основанный на принципе "pull" (вытягивание работы). Работа организована в колонки по статусам, и каждая карточка имеет четкое место в процессе.

**Преимущества Канбана:**
- ✅ **Видимость** — сразу видны все обращения и их статусы
- ✅ **Контроль** — легко отследить прогресс каждого обращения
- ✅ **Балансировка** — видно, где есть узкие места (перегруженные колонки)
- ✅ **Оптимизация** — поиск проблем и улучшение процесса
- ✅ **Скорость** — быстрое переключение между обращениями

## 🎯 Структура Канбана в TechSupport Pro

В системе 4 основные колонки статусов:

### 1️⃣ **В РАБОТЕ** (In Progress) 🟨
- **Статус:** `in_progress`
- **Цвет:** Жёлтый/Оранжевый
- **Описание:** Обращение активно обрабатывается инженером
- **Действия:** Начал работу, проводит диагностику или устранение проблемы
- **Переход:** → В Ожидании (ждём ответа клиента)  
              → Решено (проблема устранена)  
              → Не решено (не удалось решить)

### 2️⃣ **В ОЖИДАНИИ** (On Hold) ⚪
- **Статус:** `on_hold`
- **Цвет:** Серый
- **Описание:** Обращение приостановлено
- **Причины:**
  - Ожидание ответа/информации от клиента
  - Ожидание получения запасных частей
  - Ожидание результатов диагностики
- **Переход:** → В Работе (продолжить после ожидания)  
              → Решено (если оказалось, что решено)

### 3️⃣ **РЕШЕНО** (Solved) 🟢
- **Статус:** `solved`
- **Цвет:** Зелёный
- **Описание:** Проблема успешно устранена
- **Требуется:**
  - Описание принятых мер
  - Подтверждение клиента (если доступно)
- **Свойства:** Завершённое обращение архивируется

### 4️⃣ **НЕ РЕШЕНО** (Unsolved) 🔴
- **Статус:** `unsolved`
- **Цвет:** Красный
- **Описание:** Так и не удалось решить проблему
- **Причины:**
  - Клиент откз от дальнейшего сотрудничества
  - Техническая невозможность решения
  - Требуется замена оборудования целиком
- **Переход:** → В Работе (переработка/новый подход)

## 🎮 Как использовать Канбан

### Включение Канбан-вида

1. Перейдите в раздел **"Журнал обращений"** (вкладка в сайдбаре)
2. Нажмите кнопку **"Канбан"** (рядом с "Список")
3. Вы видите 4 колонки с обращениями

```
┌─────────────────┬─────────────────┬─────────────────┬─────────────────┐
│  В РАБОТЕ (5)   │ В ОЖИДАНИИ (2)   │  РЕШЕНО (12)    │ НЕ РЕШЕНО (1)   │
├─────────────────┼─────────────────┼─────────────────┼─────────────────┤
│ 🎫 #101         │ 🎫 #45          │ ✅ #89          │ ❌ #120         │
│ Принтер не ..   │ Ждет клиента    │ Камера ..       │ Проблема не ..  │
│ Engineer: John  │ Engineer: Jane  │ Engineer: Mike  │ Engineer: Sarah │
│ 09:30 вчера     │ 14:15 сегодня   │ 18:42 3 дн назад│ 11:00 сегодня   │
└─────────────────┴─────────────────┴─────────────────┴─────────────────┘
```

### Перемещение обращения между колонками

#### Способ 1: Drag & Drop (Рекомендуется)

```
1. Найдите карточку обращения, которое нужно переместить
2. Наведите мышь на карточку — она подсветится
3. Нажмите и удерживайте левую кнопку мыши
4. Перетащите карточку в нужную колонку
5. Отпустите кнопку мыши
6. ✅ Статус обновлён! Вам придёт уведомление
```

#### Способ 2: Открыть и Отредактировать

```
1. Кликните на карточку (любое место)
2. Откроется форма редактирования обращения
3. Найдите поле "Статус"
4. Выберите новый статус из 4 вариантов
5. Нажмите "Сохранить запись"
```

### Просмотр деталей обращения

```
1. Кликните на карточку обращения
2. Откроется полная информация:
   - Описание проблемы
   - Контактная информация
   - Назначенный инженер
   - История работ (если есть)
   - Статус техподдержки (гарантия/платная)
3. Если нужно, отредактируйте информацию
4. Нажмите "Сохранить запись"
```

### AI-Резюме обращения

```
1. На каждой карточке есть иконка ✨ (Sparkles)
2. Кликните на неё
3. Система сгенерирует краткое резюме проблемы и рекомендации
4. Удобно для быстрого ознакомления
```

## 💡 Лучшие практики Канбана

### ✅ Что нужно делать:

1. **Регулярно обновляйте статусы**
   - Переместите обращение сразу же при изменении статуса
   - Не забывайте про обращения "В ожидании"

2. **Ограничивайте WIP (Work In Progress)**
   - Не открывайте слишком много обращений одновременно
   - Сначала завершите текущие, потом открывайте новые

3. **Используйте поиск и фильтры**
   - Фильтр по клиенту, категории, статусу
   - Быстрая навигация по нужным обращениям

4. **Левая сторона = Начало, Правая = Конец**
   - "В работе" слева — это начало процесса
   - "Решено/Не решено" справа — это конец

5. **Проверяйте "В ожидании" регулярно**
   - Колонка "В ожидании" имеет тенденцию расти
   - Проверяйте есть ли ответы от клиентов

### ❌ Чего избегать:

1. ❌ **Оставляйте обращения в "В работе" слишком долго**
   - Если действительно завершено — переместите в "Решено"

2. ❌ **Не создавайте много обращений сразу**
   - Это затрудняет распределение работы
   - Фокусируйтесь на качестве решения

3. ❌ **Забывайте про "В ожидании"**
   - Проверяйте эту колонку ежедневно
   - Большое количество здесь = проблема c коммуникацией

4. ❌ **Не обновляйте обращение много раз в день**
   - Стабильная информация лучше, чем частые изменения

## 📈 Метрики и Аналитика

### Что может показать Канбан:

| Сигнал | Проблема | Решение |
|--------|----------|---------|
| Много в "В работе" | Перегруженность | Нанять помощника или автоматизировать |
| Много в "В ожидании" | Медленная коммуникация | Улучшить способы связи с клиентами |
| Много в "Не решено" | Сложные проблемы | Обучение или консультация специалиста |
| Медленное движение | Узкие места | Проанализировать процесс |

### Дашборд Аналитики

```
Смотрите вкладку "Дашборд" для:
- Количество обращений по статусам
- Среднее время решения (TTR)
- Статистика по категориям
- Тренды по дням/неделям
```

## 🔧 Технические детали

### Drag & Drop

- **Технология:** HTML5 Drag and Drop API
- **Браузеры:** Все современные (Chrome, Firefox, Safari, Edge)
- **Мобильные:** Поддержка прокрутки, но D&D может быть менее удобен

### Синхронизация сервера

- При перемещении карточки статус **сразу обновляется на сервере**
- Если ошибка — вы увидите toast-уведомление
- Все инженеры видят обновление в реальном времени (требует обновления страницы)

### Фильтры в Канбане

- Фильтры работают независимо для каждого вида (таблица/канбан)
- Можно искать обращения по:
  - Клиенту
  - Статусу
  - Категории проблемы
  - Дате обращения

## 🎓 Примеры использования

### Пример 1: Типичный рабочий день

```
09:00 - Приходите на работу
        Смотрите Канбан - видите распределение работ
        
09:15 - 10 обращений в "В работе"
        Выбираете с наивысшим приоритетом
        Перемещаете в "В работе" (если ещё не там)
        
10:30 - Закончили диагностику одного обращения
        Ждём ответа клиента
        Перетаскиваем в "В ожидании"
        
11:00 - Ещё одно обращение закончили
        Перемещаем в "Решено"
        
12:00 - Проверяем "В ожидании"
        Видим 2 ответа от клиентов
        Перемещаем обратно в "В работе"
```

### Пример 2: Управление нагрузкой

```
Видите что в "В работе" 15 обращений
Это слишком много для 1 инженера

Решение:
- Переместите 5 обращений в "В ожидании" (ожидаем ответов)
- Переместите 3 обращения в "Не решено" (невозможно решить)
- Осталось 7 активных - это нормальный объём
- Распределите между инженерами
```

## 🆘 Часто задаваемые вопросы

### Q: Что если я случайно переместил обращение?
**A:** Просто переместите его обратно или откройте и измените статус в форме.

### Q: Почему не обновляется после перемещения?
**A:** Подождите пару секунд. Если не обновилось - обновите страницу (F5).
Если ошибка - посмотрите toast-уведомление внизу экрана.

### Q: Могу ли я переместить закрытое обращение?
**A:** Нет, обращения в "Решено" и "Не решено" блокированы для перемещения инженерами.
Только администраторы могут их открыть повторно.

### Q: Что означает количество в скобках (5)?
**A:** Это количество обращений в этой колонке. (5) = 5 обращений в статусе.

### Q: Как фильтровать в канбане?
**A:** Используйте панель "Фильтры" над канбаном - она работает так же как в таблице.
