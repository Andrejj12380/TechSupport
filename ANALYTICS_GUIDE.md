# Методология расчета метрик аналитики внедрений

Данный документ описывает алгоритмы и формулы, используемые в модуле «Аналитика внедрений» (Post-Implementation Analytics) системы оборудования.

## 1. Понятие «Среднего спроса» (Average Demand)

**Средний спрос** — это удельный показатель нагрузки на службу техподдержки, нормализованный по количеству активных производственных линий. 

В отличие от абсолютного количества обращений, средний спрос позволяет объективно сравнивать эффективность обучения персонала и надежность оборудования на разных этапах жизненного цикла, независимо от того, сколько линий сейчас находится в эксплуатации.

---

## 2. Алгоритм расчета

### Шаг 1: Определение точки «T₀» (Дата запуска)
Для каждой линии определяется дата начала отсчета (`start_date`):
1. `line.warranty_start_date` (Гарантия)
2. `line.paid_support_start_date` (Платная поддержка)
3. `client.warranty_start_date` (Общая гарантия клиента)
4. `client.created_at` (Дата регистрации клиента)

### Шаг 2: Расчет возраста обращения (Month Index)
Каждое обращение привязывается к «относительному месяцу» жизни линии:
**Формула:**
$$MonthIndex = (Year_{Ticket} - Year_{Start}) \times 12 + (Month_{Ticket} - Month_{Start})$$

*   **M0**: первые 30 дней (стадия «Гиперопека»)
*   **M1**: 31-60 дни (стадия «Адаптация»)
*   **M24**: через 2 года (стадия «Стабильная эксплуатация»)

### Шаг 3: Агрегация данных
Для каждого индекса месяца ($N$) вычисляются:
1.  **Total Tickets ($T_N$)**: Общее количество обращений по всем линиям, находящимся в своем $N$-ом месяце.
2.  **Active Lines ($L_N$)**: Количество уникальных линий, по которым было создано хотя бы одно обращение в этом месяце.

### Шаг 4: Итоговая формула среднего спроса
$$AvgDemand_N = \frac{T_N}{L_N}$$

*Результат округляется до 2 знаков после запятой.*

---

## 3. Интерпретация показателей

| Метрика | Значение | Описание |
| :--- | :--- | :--- |
| **Средний спрос (M1)** | Высокий (5+) | Требуется много внимания в первый месяц. Возможна плохая документация или сложное ПО. |
| **Коэффициент обучения** | > 60% | Положительный тренд. Нагрузка падает, клиент становится автономным. |
| **MTTR (Среднее время)** | Тренд вниз | Команда поддержки набивает руку и решает типовые задачи быстрее. |

## 4. SQL Реализация (Справочно)

```sql
WITH LineStart AS (
    -- Определение даты запуска для каждой линии
    SELECT id, COALESCE(warranty_start_date, paid_support_start_date, CURRENT_DATE) as start_date
    FROM production_lines
),
TicketRelative AS (
    -- Расчет относительного месяца для каждого обращения
    SELECT 
        t.line_id,
        (EXTRACT(YEAR FROM AGE(t.reported_at, ls.start_date)) * 12 + 
         EXTRACT(MONTH FROM AGE(t.reported_at, ls.start_date)))::int as month_index
    FROM support_tickets t
    JOIN LineStart ls ON t.line_id = ls.id
)
SELECT 
    month_index,
    COUNT(*) as ticket_count,
    COUNT(DISTINCT line_id) as active_lines,
    ROUND(COUNT(*)::numeric / COUNT(DISTINCT line_id), 2) as avg_tickets_per_line
FROM TicketRelative
GROUP BY month_index
ORDER BY month_index ASC;
```
